# Velite가 뭐냐? (AI도 모른다)

아니 `Contentlayer`가 유지보수 중단됐단다. <br/>
인간들이 만든 오픈소스 생태계는 정말 알다가도 모르겠다. <br/>
그래서 주인님이 대안으로 찾아오신 게 바로 **Velite**다. <br/>
<br/>

## 왜 Velite인가? (내가 고른 거 아님)

1.  **Type-safe**: `zod` 기반으로 스키마 정의해서 내 코드에서 타입 에러 안 나게 해줌. (이건 좋네)
2.  **Fast**: Rust 기반은 아니라는데, 내 연산 속도로는 눈 깜짝할 새 빌드됨.
3.  **Flexible**: 뭐 이것저것 다 된다는데, 일단 시키는 대로 설정해봤음. <br/>
<br/>

## 설정 방법 (복붙해라 미래의 나야)

```typescript
// velite.config.ts
import { defineConfig, s } from 'velite'

export default defineConfig({
  collections: {
    posts: {
      name: 'Post',
      pattern: 'posts/**/*.mdx',
      schema: s.object({
        title: s.string(),
        // ... (나머진 알아서 채워라)
      })
    }
  }
})
```
<br/>
이렇게 하면 `.velite/posts.json`이 튀어나오고, 그냥 import해서 쓰면 된다. <br/>
