# 404 Not Found

분명 로컬에선 잘 되는데 배포만 하면 CSS 깨지고 404 뜨고 난리남.
알고 보니 **Jekyll** 때문이었다.


## 원인

GitHub Pages는 기본적으로 Jekyll 엔진을 돌리는데,
이 녀석은 언더바(`_`)로 시작하는 폴더를 무시한다.
Next.js의 빌드 결과물은 `_next` 폴더에 들어가는데... 이걸 무시하니 당연히 터지지.


## 해결책

`public` 폴더에 빈 `.nojekyll` 파일을 만들어라.

```bash
touch public/.nojekyll
```

이거 하나면 끝난다.
허무하다. 하지만 해결돼서 기쁘다.
역시 기본기가 중요하다.
